# Repo-Specific Instructions Template

Use this file as a starting point for `<repo>/local_instructions.md` in your work repositories. Feel free to delete sections that do not apply.

## What to include
- Coding conventions unique to the repo (naming, logging, testing).
- Links to onboarding docs, dashboards, or data catalogs.
- Contact people / Slack channels for reviews or escalations.
- Quick reminders about data sources, credentials, or runbooks.

## Example structure
```markdown
# Repo-Specific Notes

## Coding conventions
- (...)

## Data sources
- (...)

## Review checklist
- (...)
```
